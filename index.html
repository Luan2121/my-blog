¿Qué es la DNS? Y ¿Por qué podría causar una guerra corporativa?

25 min

La DNS es como el directorio de telefonos del internet, traduce nombres faciles a direcciones ip complejas de memorizar, ven y conoce un poco mas de esta asombrosa tecnología

Luis España

Desarrollador Web

Cada día se realizan millones de búsquedas a través de los principales navegadores del mundo como Google, los usuarios del internet pueden reconocer sus sitios favoritos por el nombre que cada uno tiene asociado el cual puede ser interpretado y reconocido fácilmente por las personas detrás de esta característica hay un sistema que se encarga de interpretar este nombre para que la red y los equipos informáticos la entiendan, esta tecnología es conocida como sistema de nombre de dominio, en resumen, cada uno de los sitios web tienen asociado una dirección IP que permite identificar al equipo donde se encuentra alojado el sitio web, esta dirección son un conjunto de números de la forma “XXX.XXX.XXX.XXX” el sistema de nombre de dominio se encarga de traducir esa dirección IP a un nombre o cadena de texto de modo que pueda ser facilmentente identificable y permite evitar problemas con la dirección IP ya que además de ser difícil de digerir esta puede cambiar sin embargo el nombre de dominio suele quedarse igual.

Esta tecnología es muy importante para la web y al momento de desarrollar una startup ya sea de tecnología o no es importante considerar que el nombre de la empresa ya pudo haber sido adquirido y como consecuencia no se podrá tener un sitio web bajo el mismo nombre, aunque existen otras alternativas ya que aunque el nombre “desarrollo-web.com” ya haya sido comprado; otro dominio como “desarrollo-web.net” o “desarrolloweb.com” puede aun estar disponible, a continuación se presenta un breve resumen de los aspectos principales acerca de esta tecnología,

¿Qué es la dns?

Cuando se estableció por primera vez la world wide web, para acceder a una pagina se solía utilizar la dirección IP de la misma sin embargo rápidamente se convirtió en un problema pues recordar esta serie de números no era una tarea fácil para todos, es ahí cuando se empieza a idear una forma mas sencilla de recordar dichas direcciones naciendo así los nombres de dominio y el sistema de nombres de dominio. Debido al alto crecimiento de la red de internet se hizo necesario crear instituciones que legislen la forma en la que se crean un nombre de dominio, la OMPI u Organización Mundial de la propiedad intelectual el cual pertenece al sistema de naciones unidas ha sido una de las compañías que se han encargado de tal tarea, la misma define a los sistemas de dominio como: “Los nombres de dominio de Internet pueden describirse como cómodos sustitutos de las direcciones numéricas de Internet. Una dirección numérica de Internet (también conocida como "dirección de protocolo Internet" o dirección IP) es un código numérico que permite la identificación de un ordenador dado, conectado a Internet. El nombre de dominio es el sustituto nemotécnico de dicha dirección que, si se escribiera en el ordenador, se convertiría automáticamente en la dirección numérica.”

Y...¿Por que la dns?

El sistema de nombres de dominio fue creado por Paul Mockapetris en colaboración con Jon Poste, de la "University of Southern California, USC"; durante esa época el servicio de dns se a instituciones académicas y al ejército de los Estados Unidos, presentes en Internet desde prácticamente su origen.

Inicialmente se debía almacenar una archivo titulado host.txt que contenía la información necesaria para traducir el nombre, este archivo tenía que estar contenido en cada uno los dispositivos conectados a la red, cuando se comenzó a usar el dns estos archivos constituían la base de datos en la cual los navegadores o programas se dirigían para encontrar la información del equipo, no obstante, con el crecimiento de la red este método se volvió impráctico y poco mantenible es así como entre 1983 y 1984 se modifica este sistema el cual funciona a través de una base de datos distribuida en la cual se encuentran almacenados la información de cada uno de los nombres de dominio y la dirección IP a la que hace referencia o apunta dicho nombre.

En esta etapa, la mejor forma de proveer "continuidad" al servicio dns era tener múltiples servidores contestando múltiples consultas. Un servidor era el maestro y los demás eran esclavos.

Unos 10 años después, se hicieron algunos ajustes mayores al protocolo DNS. Esto era una forma más dinámica de mantener los servidores al día, utilizando NOTIFY (en español notificar) y las transferencias incrementales de zona (IXFR).

Estructura de un nombre de dominio

Un gran numero de paginas web siguen una estructura en particular, por ejemplo:

Las siglas http hacen referencia al protocolo de transferencia de hipertexto o “hypertext transfer protocol” el cual es un protocolo utilizado por la Word wide web para la transferencia de archivos ya sean html, xml, entre otros y para la comunicación entre dispositivos, por otro lado las siglas www hacen referencia a la Word wide web que es un sistema interconectado de paginas publicas a las cuales se puede acceder por medio de internet, estos elementos se encuentran en todas las url de la web aunque estrictamente hablando no forman parte del nombre del dominio.

Existen tres niveles en el nombre de un dominio y a pesar de ser contra intuitivo estos se leen de derecha izquierda.

Los dominios de nivel superior o de primer nivel son limitados los más usados son “.com”, “.net” , “.edu” etc,

Luego tenemos los dominios de segundo nivel el cual es único para la empresa que los registra y no se puede repetir por ejemplo “Wikipedia”, “Facebook”, “twitter”.

Por ultimo los dominios de tercer nivel también llamado subdominios pertenecen generalmente a la misma url y redirigen a un subconjunto de la web o aplicación por ejemplo “.es”, “.us” “.tienda”, “.app” , para observar como afectan los subdominios a la red, al busca es.wikipidia.com se redirigirá a la Wikipedia en español.

El texto que continua después del signo de interrogación se denomina cadena de búsqueda o query string el cual representa un conjunto de datos llave valor que interpretan las aplicaciones y bases de datos con la habilidad de realizar operaciones en base a estos datos por ejemplo filtrar búsquedas enviar códigos o mensajes entre otros.

Tipos de nombres de dominios

Los dominios internacionales, genéricos o de primer nivel:

Representan el nivel mas alto en la jerarquía dns y se encuentran en la parte final del nombre de dominio, de acuerdo a la autoridad de números asignados en internet o la IANA por sus siglas en inglés, dominios se pueden clasificar en:

  • Infraestructurales (.arpa): fue el primer TLD que se creó con propósitos administrativos y de infraestructura.
  • Patrocinados (.jobs, .gov, .travel): estos dominios tienen un representante detrás por ejemplo un gobierno o agrupan comunidades específicas, por ejemplo .jobs se utiliza para el área de recursos humanos.
  • Comerciales o Generales (.net, .com, .edu): dominios con tres o más letras con propósitos de uso general
  • De código de país (.es .us): dominios de dos letras que representan a un país y siguen la norma ISO-3166

De acuerdo al uso o a la persona que utilice el dominio estos son:

  • .gov: Autoridades gubernamentales.
  • .int: Instituciones internacionales
  • .edu: Instituciones educativas
  • .mil: Instituciones militares.

Conclusión

Los problemas que resuelve el DNS han popularizado al sistema en la industria tecnológica y comercial a tal punto que muchos dominios han adquirido un gran valor y estos son considerados como una propiedad, se han abierto infinidad de debates sobre la integridad de los nombre de dominios, pues como se ha dicho anteriormente no pueden haber dos personas que posean el mismo dominio, empresas como Facebook además de tener adquirido el dominio “facebook.com” también han adquirido otros dominios como “Facebook.net” de modo que otras personas no se aprovechen de la compañía y su existo para indexar otro tipo de páginas a la web, existen casos judiciales que involucran el uso de nombres de dominios, por ejemplo recientemente una persona de nacionalidad argentina fue capaz de comprar el dominio de “Google.ar” de forma legal en una pagina de ventas de dominios esto ocurrió a consecuencia de un problema en la renovación del nombre del dominio sin embargo Google indica que su licencia aún no había expirado. Estas situaciones hacen que sea necesario instituciones y personas de varias ramas que administren y atiendan la dns de modo que la compra de dominios se pueda regular y no se genere un monopolio por parte de grandes corporaciones.

Los nombres de dominios dependiendo de su demanda muchas veces asociadas a la generalidad del nombre pueden llegar a valer mas que una casa o un auto de acuerdo A la empresa matriz Gannet Go el dominio cars.com fue comprado por el valor de 872 millones de dólares y es catalogado como la venta mas cara de un dominio.

Es importante conocer los aspectos de la dns al momento de iniciar un negocio digital pues puede que el nombre elegido para la empresa ya haya sido comprado como dominio y ya tenga tráfico asociado también se debe asegurar que el dominio no sea similar a nombres de compañías grandes, pues estas te opacaran a la hora de ser indexado en la web por ejemplo si tu sitio web fuera fase.com se vería opacado por Facebook, es así como la elección del nombre de dominio representa una tarea clave cuando se desarrollan sitios web, como consultores o programadores se debe aconsejar y advertir al cliente en estos aspectos.
